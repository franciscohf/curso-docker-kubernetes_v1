API Gateway con Nginx

Clase 4: Microservicios, Cache y API Gateway

Arquitectura

Cliente → NGINX Gateway :8080
    |
    +-- /api/*  → Backend API :5000
    |
    +-- /*      → Frontend :80
Todas las requests pasan por Nginx
Nginx actúa como punto de entrada único y distribuye el tráfico según la ruta.

Probar Backend API (via Gateway)

Todas las consultas pasan por el gateway en puerto 8080

Estado del Gateway

Información

Puerto Gateway: 8080

Puerto Backend: 5000 (no accesible directamente desde fuera)

Puerto Frontend: 80 (no accesible directamente desde fuera)

Beneficios:

  • Punto de entrada único
  • Servicios backend no expuestos directamente
  • Fácil agregar autenticación, rate limiting, etc.
  • Balanceo de carga entre múltiples instancias
